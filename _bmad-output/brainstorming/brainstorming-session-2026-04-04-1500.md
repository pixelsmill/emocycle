## Session Overview

**Sujet :** UX de la PWA Émotions — comment on vit l'app, comment on y entre, comment elle nous accompagne
**Objectifs :** Explorer des idées d'UX ancrées dans le guide des émotions et le cycle de vie émotionnel du livre, sans se soucier de ce qui existe. Créer quelque chose de sincère — une app qui est aussi un hommage.

## Phase 1 : Emotion Orchestra — Résultats

### Principes fondateurs

**[Tristesse #1] L'Écran-Souffle** — Premier écran quasi vide, épuré. L'espace vide est une permission d'exister sans effort.

**[Tristesse #2] La Caresse-Miroir** — Principe interactif subtil lié au curseur. L'interface réagit délicatement, comme si elle percevait la présence du visiteur. Fond de couleur changeant.

**[Tristesse #3] Anti-investissement** — Zéro effort demandé à l'entrée. Pas de formulaire, pas d'inscription. L'app donne avant de demander.

**[Tristesse #4] L'Émotion-Messager** — Idée centrale : une émotion n'est pas un problème, c'est un message à écouter. L'UX entière orientée écoute, pas contrôle.

**[Tristesse #5] Le Cycle Interrompu** — L'app rend visible le phénomène d'interruption du cycle émotionnel, sans moraliser.

**[Tristesse #6 révisé] Le Cycle-Présent** — Une seule émotion à la fois, celle du moment. Un cercle (le cycle en cours), pas une pile. Quand l'émotion est traversée, le cercle se complète.

### Entrée dans l'app

**[Tristesse #7] Le Pouls Chromatique** — Le fond pulse entre des couleurs associées aux familles émotionnelles. L'utilisateur clique quand une couleur résonne.

**[Tristesse #8] La Couleur-Dialogue** — La couleur choisie envahit l'écran et évolue en nuance au fil de l'exploration.

**[Tristesse #9 révisé] Couleur → Liste** — Une porte d'entrée : couleur qui pulse → liste d'émotions associées. Pas d'IA, pas d'écriture libre. Deux gestes suffisent.

**[Général #10] Promesse + Guidance** — Le premier texte est une promesse qui contient l'instruction. Ex: "Chaque émotion a un message pour vous. Laissez défiler les couleurs, et touchez celle qui vous parle."

**[Général #11] Le Miroir Calme** — L'app reste neutre quelle que soit l'émotion choisie. La stabilité EST l'empathie.

**[UX Mobile #12] Les Mots Vivants** — Liste sur fond dégradé. En scrollant, les mots grossissent au centre et diminuent en s'éloignant. Le choix est physique, tactile.

### Le cycle émotionnel

**[Cycle #13] L'Émotion Qui Parle** — "Votre [émotion] a quelque chose à vous dire." L'émotion est le sujet actif.

**[Contenu #14] La Fiche Larivey** — Après le choix, la définition du guide apparaît (exemples, description, utilité). Contenu Larivey en hommage direct.

**[Cycle #15] Le Préambule du Cycle** — "Pour recevoir ce message, laissez-vous traverser par chaque étape."

**[Cycle #16] Émergence = Accueil** — "Cette émotion est là. Laissez-la prendre sa place." Accepter, se concentrer.

**[Cycle #17] Reconnaissance = Le Besoin Caché** — "Un de vos besoins demande votre attention. C'est une chance de le comprendre." Le concept clé, présenté comme une bonne nouvelle.

**[Cycle #18] Le Cycle Bifurque** — Deux branches selon positif/négatif : besoin comblé vs insatisfait. Classification des émotions pour adapter le langage.

**[Cycle #19 révisé] Les Deux Voix du Cycle** — Étapes 1-2 universelles, bifurcation à partir de l'étape 3 :
- Insatisfait : "Et si cette émotion était liée à un besoin qui vous appelle ?"
- Comblé : "Et si cette émotion vous montrait ce qui compte vraiment pour vous ?"
- Étape 5 sans présupposé de personne concernée, avec exemples variés.

**[Contenu #20] Classifier le Guide** — Pré-classification positif/négatif des ~80 émotions. Question de repli pour les ambiguës : "agréable ou désagréable en ce moment ?"

**[Cycle #21] Les Exemples-Guides** — Exemples discrets sous chaque question du cycle pour débloquer sans diriger.

### Fin de parcours

**[Fin #22] La Sortie Douce** — "Nous espérons que cette expérience va vous aider à avancer. Prenez soin de vous. Revenez quand vous voulez." + [Merci] / [Pas vraiment]

**[Fin #23 révisé] Le "Pas vraiment" — Humble** — "Ce n'est pas grave. Vous avez eu le courage d'essayer, et c'est déjà beaucoup." Pas de promesse que ça marchera la prochaine fois.

**[Fin #24] Stats pour l'admin** — Les deux boutons alimentent des stats anonymes côté admin.

**[Fin #25] Le Filet de Sécurité** — Lien discret vers numéros d'écoute. L'app sait qu'elle n'est pas une thérapie.

### Architecture de contenu

**[Contenu #26] Les Fiches Nourrissent le Cycle** — Extraire des passages des fiches pour alimenter chaque étape du cycle (sensations, "à quoi ça sert", erreurs typiques).

**[Contenu #27] La Classification Existe Déjà** — Le guide fournit la classification positif/négatif dans simple.html. Pré-mappable.

**[Contenu #28] Les Erreurs Typiques** — Certaines fiches décrivent les interruptions de cycle. "Attention, il arrive souvent de..." — texte d'aide consultable à tout moment.

**[Contenu #29] Les Émotions-Masques** — Après le cycle, suggérer qu'une émotion peut en cacher une autre. Invitation à refaire un tour.

**[Contenu #30] Le Fallback Bienveillant** — Texte générique orienté positif/négatif quand une fiche manque de contenu pour une étape.

**[UX #31] La Page "Comprendre le Cycle"** — Texte accessible à tout moment : le cycle, la tentation de l'interrompre, chaque étape décrite. Pas un prérequis.

**[Contenu #32] Les 4 Types = 4 Parcours** — Simple → cycle standard. Mixte → démêler les émotions composantes. Repoussée → retrouver l'émotion cachée. Pseudo → identifier la vraie émotion dessous. Tous redirigent vers le cycle d'une émotion simple.

**[UX #33] Le Pouls Tranquille** — Rotation lente de la teinte (0-360° HSL), saturation et luminosité fixes. Rythme doux comme une respiration.

**[UX #34] La Sortie de Secours "Autre chose"** — Si les émotions listées ne correspondent pas après le choix de couleur, lien "Non, je ressens autre chose" → accès au dictionnaire complet des ~80 émotions.

## Phase 3 : Structure de données

### Analyse du contenu des fiches

Le guide contient ~80 fiches HTML (1998-2005, encodage latin1). Structure récurrente :
- Titre + type (simple/mixte/repoussée/pseudo)
- Synonymes / expériences connexes
- Exemples (2-4 situations concrètes)
- "Qu'est-ce que c'est ?" (définition)
- "À quoi ça sert ?" (utilité, lien avec les besoins)
- Optionnel : sensations physiques, erreurs typiques, confusions entre émotions

Le contenu varie fortement : colère/tristesse/peur sont riches, contentement/boule sont courts.

### Classification existante (simple.html)

**Positives (besoin comblé) :**
- Par rapport au besoin : joie, plaisir, contentement, émerveillement...
- Par rapport au responsable : amour, fierté, tendresse...
- D'anticipation : désir, envie, excitation...

**Négatives (besoin insatisfait) :**
- Par rapport au besoin : tristesse, ennui, nostalgie...
- Par rapport au responsable : colère, rage, haine, dégoût...
- D'anticipation : peur, terreur, effroi...

### Idées structure de données

**[Data #35] Le Pouls Ne Montre Que les Familles Simples** — Les 6 couleurs du pouls = 6 familles d'émotions simples. Mixtes/repoussées/pseudo accessibles via "Autre chose".

**[Data #36] Le Fallback Structuré** — Texte générique par valence pour chaque étape du cycle quand la fiche ne fournit pas le contenu.

**[Data #37] Le Parsing Réaliste** — Extraction ponctuelle HTML → markdown, puis structuration JSON. Pas de parser temps réel.

**[Data #38] Modèle en Anglais** — Toutes les propriétés du modèle en anglais :
```
Emotion { slug, title, type, valence, family, hue, synonyms, examples,
          definition, purpose, sensations, mistakes, confusions,
          relatedEmotions, hiddenEmotions, components }
```

**[Data #39] Extraction Markdown d'Abord** — Avant le JSON, extraire le texte brut de chaque fiche HTML dans un {slug}.md propre. Source de vérité humaine.
